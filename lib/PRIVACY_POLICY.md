# Политика конфиденциальности

**Дата вступления в силу:** 9 февраля 2026 г.
**Версия:** 1.0.0

---

## 1. Общие положения

Настоящая Политика конфиденциальности описывает, как приложение "СообщиО" собирает, использует и защищает персональные данные пользователей.

---

## 2. Собираемые данные

### 2.1. Персональные данные
Приложение может собирать следующие данные:

1. **Данные Telegram аккаунта** (при авторизации через Telegram):
   - Имя пользователя
   - Фамилия
   - ID пользователя (Telegram ID)
   - Аватар пользователя
   - Имя пользователя (@username)
   - Язык

2. **Жалобы пользователей:**
   - Заголовок жалобы
   - Описание проблемы
   - Категория проблемы
   - Местоположение (широта, долгота)
   - Адрес (если указан)
   - Статус жалобы
   - Дата создания
   - Фотографии (если приложены)

3. **Местоположение:**
   - Координаты GPS (ширина, долгота)
   - Адрес (при ручном вводе или обратном геокодинге)

4. **Технические данные:**
   - FCM токен для push-уведомлений
   - ID устройства
   - Версия приложения
   - Операционная система

### 2.2. Данные, которые НЕ собираются
Приложение НЕ собирает и НЕ хранит:
   - Пароли (используется только хешированный токен)
   - История просмотра
   - История поиска
   - Метаданные фотографий (кроме EXIF)
   - История нажатий (если не требуется для функционала)

---

## 3. Цели обработки данных

Собранные данные используются для:
- Создания и отслеживания жалоб
- Отправки push-уведомлений о статусе жалоб
- Геолокации проблем на карте
- Статистики и аналитики
- Оптимизации пользовательского опыта

---

## 4. Правила сбора данных

### 4.1. Ввод данных
Персональные данные вводятся пользователем добровольно:
- Через авторизацию Telegram (нажатие кнопки "Войти через Telegram")
- При создании жалобы (заголовок, описание, категория)
- При добавлении фотографии к жалобе

### 4.2. Местоположение
Геолокация используется только для:
- Отображения жалобы на карте
- Определения ближайших проблем
- Отправки жалобы в соответствующие службы

Геолокация запрашивается только:
- При создании жалобы
- При использовании функции "Моя локация"

### 4.3. Фотографии
Фотографии собираются только при:
- Согласии пользователя на публикацию
- Создании новой жалобы
- Фотографии не используются для других целей

---

## 5. Хранение данных

### 5.1. Локальное хранилище (Hive)
На устройстве пользователя хранится:
- ID пользователя
- Токен авторизации
- Локальные черновики жалоб
- Настройки приложения

Локальное хранилище зашифровано с использованием:
- Hive encryption
- Ключ шифрования уникален для каждого устройства

### 5.2. Облачное хранилище (Backend)
На сервере хранится:
- Все созданные жалобы
- Данные пользователей
- Статистика
- FCM токены для push-уведомлений

Облачное хранилище защищено:
- SSL/TLS шифрование при передаче данных
- Ограничение доступа по IP-адресам
- Регулярное резервное копирование

### 5.3. Сроки хранения
- Жалобы хранятся бессрочно
- Неактивные пользователи удаляются через 12 месяцев
- FCM токены обновляются при каждом входе

---

## 6. Использование данных

### 6.1. Передача третьим лицам
Приложение НЕ передает данные третьим лицам, за исключением:
- **Геокодинг:** Для получения адреса по координатам используется Nominatim (бесплатный сервис)
- **Push-уведомления:** Отправляются через Firebase Cloud Messaging
- **Хостинг:** Backend размещен на защищенном сервере

### 6.2. AI анализ
Текст жалоб анализируется с использованием:
- Zai GLM-4.7 (локальная модель)
- Текст отправляется на сервер, где проходит анализ
- Результат анализа (категория, адрес, саммари) возвращается приложению

### 6.3. Аналитика и статистика
Приложение может собирать анонимные данные:
- Количество просмотренных жалоб
- Количество созданных жалоб
- Время использования приложения
- Статистика по категориям жалоб

Эти данные используются для:
- Улучшения качества сервиса
- Оптимизации производительности
- Персонализации контента

---

## 7. Безопасность данных

### 7.1. Шифрование
- Все данные передаются по HTTPS
- Токены авторизации хешируются
- Локальное хранилище зашифровано
- База данных защищена паролями

### 7.2. Авторизация
- Используется OAuth 2.0 через Telegram
- Токены имеют срок действия
- Токены можно отозвать в любое время
- При каждом входе создается новый токен

### 7.3. Защита от несанкционированного доступа
- Проверка токена при каждом запросе
- Ограничение количества запросов
- Блокировка подозрительной активности

---

## 8. Права пользователя

### 8.1. Доступ к данным
Пользователь имеет право:
- Просматривать свои жалобы
- Редактировать свои жалобы
- Удалять свои жалобы
- Получать уведомления о статусе своих жалоб
- Отозвать согласие на использование данных

### 8.2. Экспорт данных
Пользователь может запросить:
- Все свои персональные данные в формате JSON
- Историю всех своих жалоб
- Удаление всех своих данных и аккаунта

Для экспорта или удаления данных необходимо:
- Авторизоваться в приложении
- Нажать "Экспорт данных" или "Удалить аккаунт"
- Подтвердить действие

---

## 9. Политика в отношении детей

Приложение НЕ предназначено для детей младше 13 лет.
При обнаружении использования приложения лицом моложе 13 лет:
- Доступ к приложению будет ограничен
- Данные будут удалены в течение 24 часов

---

## 10. Cookies и другие технологии отслеживания

Приложение НЕ использует:
- Cookies для отслеживания
- Локальное хранилище браузера (Web)
- Fingerprinting или Canvas Fingerprinting

Приложение использует:
- Локальное зашифрованное хранилище (Hive)
- Secure Storage для чувствительных данных
- Firebase Cloud Messaging для push-уведомлений

---

## 11. Уведомления об изменениях

Пользователи будут уведомлены об изменениях в настоящей Политике:
- Через push-уведомления в приложении
- Через email (при наличии аккаунта)
- Через Telegram bot (при наличии)

Изменения вступают в силу через 30 дней после уведомления.
