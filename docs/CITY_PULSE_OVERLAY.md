# Оверлей пульса города на карте

## Описание

Визуализация "пульса города" в виде оверлея на карте, который реагирует на появление жалоб и происшествий в реальном времени. Пульс изменяется в зависимости от категории жалобы и уровня общественной опасности.

## Функциональность

### 1. Визуализация пульса

**Оверлей на карте:**
- Canvas overlay поверх карты (`#cityPulseOverlay`)
- Концентрические круги от центра карты
- Пульсация зависит от BPM (ударов в минуту)
- Цветовая индикация состояния города

**Волновая форма:**
- Визуализация внизу экрана
- Отображает историю пульсации
- Органичная форма волны

### 2. Реакция на жалобы

**При появлении новой жалобы:**
- Создается событие пульсации (`pulseEvent`)
- Интенсивность зависит от уровня опасности категории
- Визуальный эффект: концентрические круги от места жалобы
- Эффект длится 3 секунды с затуханием

**Координаты жалобы:**
- Если есть координаты (lat/lng), пульсация происходит в месте жалобы
- Иначе используется случайная позиция или центр карты

### 3. Система оценки опасности

**Уровни опасности по категориям:**

| Уровень | Категории | Описание |
|---------|-----------|----------|
| **5** | Безопасность, ЧП, Газоснабжение | Критическая опасность |
| **4** | Медицина, Отопление | Высокая опасность |
| **3** | Дороги, Снег/Наледь, Водоснабжение | Средняя опасность |
| **2** | ЖКХ, Транспорт, Экология, Освещение, Образование | Низкая опасность |
| **1** | Благоустройство, Парковки, Прочее | Минимальная опасность |

### 4. Расчет BPM (ударов в минуту)

**Формула:**
```
BPM = 60 + (средняя_опасность × 8) + (количество_событий × 2)
Максимум: 150 BPM
```

**Диапазоны:**
- **60-70 BPM**: Спокойно (зеленый)
- **70-90 BPM**: Умеренно (желтый)
- **90-120 BPM**: Напряженно (оранжевый)
- **120-150 BPM**: Тревожно (красный)

### 5. Цветовая индикация

**Цвета пульса:**
- **Зеленый** (BPM < 70): `rgba(45, 212, 191, 0.3)` - Спокойно
- **Желтый** (BPM < 90): `rgba(245, 158, 11, 0.4)` - Умеренно
- **Оранжевый** (BPM < 120): `rgba(249, 115, 22, 0.5)` - Напряженно
- **Красный** (BPM ≥ 120): `rgba(239, 68, 68, 0.6)` - Тревожно

## Технические детали

### Компонент CityPulseOverlay

**Основные методы:**

1. **`init()`** - Инициализация canvas overlay
2. **`reactToComplaint(complaint)`** - Реакция на новую жалобу
3. **`updateBPM()`** - Обновление BPM на основе событий
4. **`getPulseColor()`** - Получение цвета на основе BPM
5. **`animate()`** - Основной цикл анимации
6. **`drawWaveform(ctx, W, H)`** - Рисование волновой формы

### Интеграция с real-time обновлениями

**При появлении новой жалобы:**
```javascript
// В функции checkForNewComplaints()
newComplaints.forEach(complaint => {
  addMarkerWithAnimation(complaint);
  CityPulseOverlay.reactToComplaint(complaint);
});
```

**При загрузке данных:**
```javascript
// В функции loadData()
state.complaints.forEach(complaint => {
  CityPulseOverlay.reactToComplaint(complaint);
});
```

### Структура события пульсации

```javascript
{
  time: Date.now(),           // Время создания
  intensity: 0.0-1.0,        // Интенсивность (dangerLevel / 5)
  category: "Безопасность",  // Категория жалобы
  dangerLevel: 5,            // Уровень опасности (1-5)
  lat: 60.9344,              // Широта (опционально)
  lng: 76.5531,              // Долгота (опционально)
  x: null,                   // X координата на экране
  y: null                    // Y координата на экране
}
```

## Визуальные эффекты

### Основной пульс
- Концентрические круги от центра карты
- Радиус: `baseRadius + sin(phase) × intensity × 2`
- Градиент с затуханием к краям
- Mix-blend-mode: screen для эффекта свечения

### События от жалоб
- Концентрические круги от места жалобы
- Радиус увеличивается со временем (30px + age × 20px)
- Прозрачность уменьшается со временем (fade out за 3 секунды)
- Интенсивность зависит от уровня опасности

### Волновая форма
- Отображается внизу экрана (высота 60px)
- История из 200 точек
- Органичная форма с шумом
- Цвет соответствует текущему BPM

## Производительность

**Оптимизации:**
- Ограничение количества событий (максимум 50)
- Автоматическое удаление старых событий (> 3 секунд)
- Использование requestAnimationFrame для плавной анимации
- Прозрачность для эффекта следа вместо полной очистки

## Использование

Оверлей автоматически инициализируется при загрузке карты и начинает реагировать на жалобы в реальном времени. Никаких дополнительных действий не требуется.

## Примеры реакций

**Низкая опасность (Благоустройство):**
- Слабая пульсация
- Зеленый цвет
- BPM увеличивается незначительно

**Средняя опасность (Дороги):**
- Умеренная пульсация
- Желтый/оранжевый цвет
- BPM увеличивается заметно

**Высокая опасность (Безопасность, ЧП):**
- Сильная пульсация
- Красный цвет
- BPM резко увеличивается
- Яркие визуальные эффекты
