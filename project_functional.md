1. Сити‑Мониторинг (Сбор данных в реальном времени)
Модуль агрегирует проблемы горожан из локальных новостных пабликов, исключая необходимость ручной обработки.

Telegram‑сканирование (telegram_monitor.py)
Автоматическое чтение и прослушивание сообщений из ключевых каналов города (например, nizhnevartovsk_chp, adm_nvartovsk).

VKontakte‑опросы (vk_monitor_service.py)
Мониторинг групп и пабликов ВКонтакте для выявления обращений и упоминаний городских проблем.

Антиспам и дедупликация (realtime_guard.py)
Интеллектуальный фильтр отбраковывает рекламу, вакансии и спам.
Если об одной и той же проблеме написали пять разных людей в радиусе 100 метров, система распознаёт это как единый инцидент.

2. Мозг Системы (AI Analyzer)
Переводит неструктурированный текст горожан в четкий служебный отчёт.

Анализ текста (zai_service.py, ai_proxy_service.py)
С применением LLM (Zai GLM‑4.7 / Claude) извлекается:

Категория: ЖКХ, Дороги, Благоустройство, Освещение и ещё порядка 15 видов проблем.

Суть: краткая, структурированная выжимка инцидента.

Адрес: точный адрес происшествия из контекста.

Расшифровка фото (zai_vision_service.py)
Если сообщение состоит лишь из картинки (например, «ямы на дороге»), встроенный Vision‑модуль распознаёт тип проблемы по изображению.

Метаданные (EXIF) (exif_service.py)
Извлекает скрытые GPS‑координаты из загруженных фотографий для точной привязки к месту события.

3. Геоинформационная система (GIS)
Привязывает текстовый адрес к конкретным координатам на карте.

Геокодирование (geo_service.py)
Конвертация извлечённого нейросетью адреса в пару координат (широта, долгота).

Формирование ссылок на карты
Автоматическая генерация кликабельных ссылок на Google Maps или Street View для визуального осмотра места инцидента.

4. Серверная часть (Бэкенд на FastAPI)
Модуль для управления API и открытыми данными.

Асинхронные интерфейсы
Набор безопасных endpoint’ов (шлюзов) для внешних запросов, включая мониторинг, экспорт данных и интеграцию с клиентскими приложениями.

Поддержка открытых данных (opendata.py)
Аккумулирует реестры управляющих компаний и других ответственных организаций.

Сервис ответственности (uk_service.py)
По координатам инцидента определяется, кто несёт ответственность за конкретный дом или двор (УК, МО, администрация и т.п.).

5. Гражданский бот (Ultimate Bot)
Прямой интерфейс для общения с жителями (ultimate_bot.py).

Возможность отправлять жалобы напрямую в систему: текст, фото, голосовые сообщения.

Очистка голосовых сообщений (vocal_remover_service.py)
Применение передовой фильтрации для подавления уличного шума (ветер, трасса), чтобы максимально точно распознать речь горожанина.

6. Единый канал‑агрегатор и хранилище
Весь результат сводится к быстрому и эффективному реагированию.

Единый служебный канал
Все чисто отработанные инциденты публикуются в служебный канал (например, @monitornv) в стандартизированном формате:
категория, выжимка, адрес, координаты карты и кликабельный источник.

Локальное и облачное хранилище

Локально: SQLite‑база (soobshio.db).

Облачно: реальновременная синхронизация через supabase_service.py для облачных клиентов и аналитики.

Мобильная интеграция / Карта
В папке Frontend (на Flutter) находится интерактивная карта, на которую в реальном времени проецируются все инциденты города.
Город смотрит на проблемные зоны глазами «Пульса города».
